- ### 项目概述：基于 FreeRTOS 的平衡平衡小车 (STM32F103C8T6)

  #### 1. 项目目标

  基于 **STM32F103C8T6**（20KB RAM, 64KB Flash）开发一款两轮自平衡小车。使用 **FreeRTOS** 进行任务调度，实现小车直立行走、蓝牙远程控制及超声波自动避障功能。**CubeMX生成的初始化代码，有以下注意事项：**

  #### 2. 硬件资源分配 (Pin Mapping)

  - **传感器 (I2C)：** MPU6050（姿态感知：加速度计+陀螺仪）。
  - **电机驱动 (PWM)：** TB6612 驱动，控制两路直流电机。
  - **编码器 (Timer Encoder Mode)：** 两路正交编码器，用于速度反馈。
  - **显示屏 (I2C)：** 0.96寸 OLED，用于显示电池电压、姿态角和运行状态。
  - **避障 (GPIO/Timer)：** 超声波测距模块 (HC-SR04)。
  - **通信 (UART)：** 蓝牙模块，接收手机 App 或遥控器的控制指令。UART我使用了DMA，TX和RX都使用了**DMA**

  #### 3. 软件架构与任务划分 (精简双任务模型)

  考虑到 C8T6 的 RAM 资源非常有限，系统采用**“1路实时控制 + 1路辅助交互”**的设计：

  - **任务 A：核心控制任务 (Motion_Control_Task)**
    - **优先级：** 最高 (osPriorityRealtime)
    - **执行周期：** 5ms - 10ms (硬实时)
    - **功能：** 读取 MPU6050 数据 -> 姿态融合计算（滤波） -> 读取编码器速度 -> **执行三环 PID 算法（直立环、速度环、转向环）** -> 更新 PWM 输出。
    - **要求：** 必须保证定频运行，不准有阻塞操作。
  - **任务 B：系统辅助与 UI 任务 (System_UI_Task)**
    - **优先级：** 普通 (osPriorityNormal)
    - **执行周期：** 50ms - 100ms
    - **功能：** 触发超声波测距 -> 逻辑判断（如遇到障碍物修改目标速度） -> 刷新 OLED 屏幕数据 -> 电池电压监测。
    - **要求：** 负责处理耗时较长的 I2C 刷新和传感器等待逻辑，避免干扰控制任务。
  - **中断处理 (Interrupts)：**
    - **串口中断：** 接收蓝牙原始数据，解析控制命令并更新全局控制目标（如前进、后退、转向、调参）。

  #### 4. 数据通信与同步策略

  - **全局数据结构：** 定义一个全局结构体 Robot_Control_t，存储当前姿态、目标速度、PID 参数、超声波距离等信息。	
  - **同步：** 任务间采用轻量级的全局变量同步，减少 FreeRTOS 信号量或队列带来的内存开销。

  #### 5. 开发建议与限制

  - **库文件：** 使用 STM32 HAL 库 + FreeRTOS(FreeRTOSv202212.00)。
  - **内存优化：**
    - 控制任务堆栈分配约 1024 字节（涉及浮点运算）。
    - 辅助任务堆栈分配约 512 字节。
    - 关闭不必要的 FreeRTOS 功能以节省 Flash/RAM。
  - **分层设计：** 代码需分为 BSP 驱动层、PID/滤波算法层、以及 RTOS 任务应用层。
