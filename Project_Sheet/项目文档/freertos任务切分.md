### 1. 任务切分方案

#### **任务一：平衡控制核心任务 (Control_Task)**

- **优先级：** osPriorityRealtime (最高)
- **周期：** 5ms 或 10ms (硬实时，严禁延时)
- **负责内容：**
  1. **传感器读取：** 读取 MPU6050 数据。
  2. **姿态解算：** 互补滤波或卡尔曼滤波。
  3. **编码器反馈：** 读取左右轮编码器数值。
  4. **PID 计算：** 依次执行直立环、速度环、转向环。
  5. **动作执行：** 更新定时器 PWM 输出给电机驱动。
- **理由：** 平衡车的核心逻辑必须高度同步。将 MPU 读取和 PID 放在一起，可以减少任务切换开销，并保证控制频率的绝对稳定。

#### **任务二：系统辅助与交互任务 (System_Task)**

- **优先级：** osPriorityNormal (普通)
- **周期：** 50ms - 100ms (软实时)
- **负责内容：**
  1. **避障检测：** 触发超声波测距（非阻塞或短时间等待）。
  2. **状态显示：** 刷新 OLED 屏幕（显示电量、姿态等）。
  3. **逻辑判断：** 根据避障结果或蓝牙指令，修改任务一中的 PID 目标值。
  4. **电池监测：** 检测电压，防止锂电池过放。
- **理由：** OLED 刷新和超声波等待非常耗时，必须从核心控制任务中剥离，否则车会瞬间倒下。

#### **隐藏“任务”：串口中断服务 (UART_IRQHandler)**

- **负责内容：** 接收蓝牙指令。
- **做法：** 采用串口中断接收，解析出指令后直接修改全局变量（如 Target_Speed）。不需要单独开任务，节省堆栈。



### 2. 任务间通信（最省资源方式）

对于 C8T6 这种只有 20KB RAM 的芯片，尽量少用 Queue (队列)，多用 **全局变量 + 标志位**：

- **数据共享：** 定义一个全局结构体 Control_Params，包含目标速度、转向角度等。
- **任务二 -> 任务一：** 任务二处理完避障逻辑后，直接修改 Control_Params。
- **任务一 -> 任务二：** 任务一计算出的姿态数据存入全局变量，供任务二在 OLED 上显示。



### 3. C8T6 堆栈分配建议 (configTotalHeapSize 设为 10-12KB 左右)

| 任务名称            | 堆栈大小 (Stack Size)  | 说明                                 |
| ------------------- | ---------------------- | ------------------------------------ |
| **Control_Task**    | 256 Words (1024 bytes) | 包含浮点运算（PID/滤波），堆栈需稍大 |
| **System_Task**     | 128 Words (512 bytes)  | 主要是 I2C 逻辑，不涉及复杂运算      |
| **Timer/Idle Task** | 64 Words               | FreeRTOS 自带，设为最小值            |
