两个摇杆，两个摇杆！！！一个摇杆是动力，一个是方向，参考玩具小车遥控的设计

## 一、命令格式总体设计

- **物理层**：UART，波特率 115200，8 位数据，1 位停止位，无校验
- **帧长度**：固定 6 字节
- **传输顺序**：小端模式（先发低字节）
- **帧结构**：

| 帧头 (1B) | 命令字 (1B) | 动力值 (1B)（kp） | 方向值 (1B)（ki） | 保留 (1B)（kd） | 校验和 (1B) |
| :-------- | :---------- | :---------------- | :---------------- | :-------------- | :---------- |
|           |             |                   |                   |                 |             |

------

## 二、字段详细说明

### 1. 帧头

- 固定值 `0xAA`，用于接收方同步帧起始。

### 2. 命令字

- `0x01`：动力+方向控制命令（常规控制）
- `0x02`：急停命令（忽略动力/方向字段）
- `0x03`：校准/复位命令
- `0x04`：调节角度环pid命令
- `0x05`：获取pid三个参数
- `0x06`：单独按照步长增加/减小pid三个参数
- `0x07`：启动/关闭pid
- `0x08`：调节速度环pid命令
- `0x09`：调节转向环pid命令

### 3. 动力值

- 范围：`0x00 ~ 0xFF`
- 映射关系：
  - `0x00`：最大后退
  - `0x7F`：停止（中点）
  - `0xFF`：最大前进
- 实际使用时，将遥控器油门摇杆的 ADC 值（例如 0~4095）线性映射到 0~255 即可。

### 4. 方向值

- 范围：`0x00 ~ 0xFF`
- 映射关系：
  - `0x00`：最大左转
  - `0x7F`：直行（中点）
  - `0xFF`：最大右转

### 5. 保留字节

- 固定为 `0x00`，可用于未来扩展（如灯光、鸣笛等辅助功能）。

### 6. 校验和

- 计算方法：对**帧头、命令字、动力值、方向值、保留字节**这 5 个字节进行累加和（模 256），然后取低 8 位。
- 即：`Checksum = (0xAA + Cmd + Throttle + Steering + Reserve) & 0xFF`

#### PID 编码与真实值的映射

为了便于玩具级控制器传输，每个系数使用 **1 字节无符号整数** 编码，平衡车内部根据以下线性映射转换为真实 PID 值：

- **Kp 真实值** = `Kp_encoded / 10.0`
  范围：0.0 ~ 25.5（步长 0.1）
  
- **Ki 真实值** = `Ki_encoded / 100.0`
  范围：0.00 ~ 2.55（步长 0.01）
  
- **Kd 真实值** = `Kd_encoded / 10.0`
  范围：0.0 ~ 25.5（步长 0.1）
  
  ### 8.单独按照步长增加/减小pid三个参数
  
  1. 动力值->0x00  0x01  0x02  分别对应kp，ki，kd
  2. 方向值->0x00减   0x01加
  3. 保留->角度环，速度环，转向环的选择分别为0x00 0x01 0x02
  
  ### 9.启动关闭pid控制
  
  1.动力值->   0x01 启动  0x00关闭

## 三、命令示例

### 例1：全速前进 + 直行

- 帧头：`0xAA`
- 命令字：`0x01`
- 动力值：`0xFF`（最大前进）
- 方向值：`0x7F`（直行）
- 保留：`0x00`
- 校验和：`(0xAA+0x01+0xFF+0x7F+0x00) = 0x229` → 低8位 `0x29`

**完整命令**：`AA 01 FF 7F 00 29`

### 例2：中点停止 + 最大右转

- 动力值：`0x7F`（停止）
- 方向值：`0xFF`（最大右转）
- 校验和：`(0xAA+0x01+0x7F+0xFF+0x00) = 0x229` → `0x29`

**完整命令**：`AA 01 7F FF 00 29`

### 例3：急停（立即停止电机）

- 帧头：`0xAA`
- 命令字：`0x02`
- 动力值/方向值/保留均忽略（填 `0x00`）
- 校验和：`(0xAA+0x02+0x00+0x00+0x00) = 0xAC`

**完整命令**：`AA 02 00 00 00 AC`

## 四、接收端处理逻辑（平衡车主控）

1. 通过 UART 中断接收字节，检测到 `0xAA` 后开始组装一帧。
2. 收满 6 字节后，计算校验和，与最后一个字节比对。
3. 若校验通过，根据命令字执行动作：
   - `0x01`：将动力值、方向值分别转换为电机目标转速和转向差速系数，输入到平衡车的自平衡 PID 控制器中。
   - `0x02`：立即停止电机，进入安全状态。
   - `0x03`：触发 IMU 校准流程。
4. 若校验失败或超时未收完，丢弃该帧并重新搜索 `0xAA`。

## 五、扩展建议

- 若需要反向控制（如后退时方向相反），可在接收端软件中根据动力值的正负自动调整方向值的极性。
- 保留字节可后续用于定义“模式切换”（如运动模式/新手模式）或“灯光控制”。
- 对于无线遥控，建议在发送端增加简单的重传机制（同一帧连续发 2~3 次），降低丢包影响。
