3. 修正方案
第一步：修正互补滤波逻辑
你应该将滤波后的结果反馈回下一轮计算。
code
C
// 1. 先计算加速度计角度
angleAcc = COMPUTE_ANGLE_ACC(mpu6050Data.AccY, mpu6050Data.AccZ);

// 2. 互补滤波： 最终角度 = 陀螺仪占比 * (上次最终角度 + 陀螺仪积分项) + 加速度计占比 * (本次加速度计角度)
// 注意：这里使用的是 angleSmooth 本身作为“上一时刻角度”
angleSmooth = (GYROSCOPE_SHARE * (angleSmooth + (mpu6050Data.GyroX / SENSITIVITY) * SAMPING_PERIOD)) 
            + (ACCELERATION_SHARE * angleAcc);

在初始化 MPU6050 时，静止放置传感器，读取 500-1000 个数据求平均值，得到 GyroX_Offset。
在计算时：
actualGyroX = mpu6050Data.GyroX - GyroX_Offset;

总结代码建议：
code
C
// 假设在初始化时已经计算出 gyroX_offset
float gyroRate = (mpu6050Data.GyroX - gyroX_offset) / 16.4f; // 转换为 °/s

// 加速度计角度计算（确保使用 float 防止截断）
float accAngle = -atan2((float)mpu6050Data.AccY, (float)mpu6050Data.AccZ) * (180.0f / PI);

// 互补滤波标准公式
// 融合比例：dt / (τ + dt), τ 是时间常数，通常取 0.1s 左右
float alpha = 0.98f; 
angleSmooth = alpha * (angleSmooth + gyroRate * 0.01f) + (1.0f - alpha) * accAngle;

// 打印
printf("[plot,%.3f]\r\n", angleSmooth);
为什么 angleAcc 一直是 0 的最后检查：
如果执行 angleAcc = -atan2(0, 16384)（静止水平放置），结果确实是 0。只有当你倾斜传感器时，AccY 才会产生数值，angleAcc 才会变化。如果你倾斜了还是 0，请检查 mpu6050Data.AccY 的读取函数是否真的拿到了原始数据。
