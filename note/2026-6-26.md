## 低通滤波算法

用于平滑编码器测速，在Moitor模块
// 滤波系数：0.0 到 1.0 之间。数值越大越平滑，但延迟越高。
#define SPEED_FILTER_ALPHA 0.5f         //目前测试只用了原始速度，没有滤波，后续可以调节这个值来实现滤波
- 低通滤波算法SPEED_FILTER_ALPHA调试：通过对比原本的波形和滤波之后的波形，主要看这个值能不能消除尖刺的同时和保持快速响应（也就是速度回到0的时候，滤波后的波形是不是很快的到0）。如果能兼顾两者，那说明这个值可以，是个不错的低通滤波算法的系数

// 架空小车（轮子悬空），用手转动轮子，观察波形图：
// 如果波形如下（ALPHA 太小，如 0.1）：
// 红线（原始）：毛刺非常多，跳动剧烈。
// 绿线（滤波）：几乎重合在红线上，毛刺依然很多。
// 结论：滤波太弱，噪声会传给 PID 导致电机震动。
// 如果波形如下（ALPHA 太大，如 0.95）：
// 红线（原始）：有毛刺。
// 绿线（滤波）：非常平滑，是一条漂亮的曲线。
// 但是延迟极大：当你用手突然刹停轮子时，红线瞬间掉到 0，而绿线像滑梯一样，过了半秒钟才慢慢降到 0。
// 结论：延迟太大，这会导致你的平衡车出现“荡秋千”现象（相位滞后导致的发散性震荡）。
// 理想状态（ALPHA 约 0.7~0.8）：
// 红线（原始）：有毛刺。
// 绿线（滤波）：大部分高频毛刺被削平了，看起来比红线顺滑很多。
// 延迟表现：当你快速拨动轮子时，绿线能紧紧跟上红线的趋势，延迟感很微弱。
// 你的目标：
// 找到一个 ALPHA 值，使得绿线足够平滑（不传递高频抖动），同时又反应够快（延迟不超过 100-200ms）。


## 在只有直立环的时候我ki不能为0，只有直立环的时候ki为0就会溜车到倒下了 加上速度环的时候就因为这个ki而前后摆动然后倒下了
1. 为什么会来回摆动？（深度诊断）
直立环 Ki： 发现车有点歪，就让电机往前跑，试图消除那个 0.1 度的偏差。
速度环： 发现电机在跑，就大喊：“不行！速度要归零！”于是强行让直立环目标角度后仰。
冲突： 你的直立环 kp只有 18.5，力量不够。而速度环 kp只有 0.09，力量太小。
结果： 它们在“温水煮青蛙”，谁也压不住谁，导致摆动幅度越来越大，最终电机 PWM 到达 100 极限，系统崩溃倒下。


## 我tt电机最大转速就是7.3啊
这是一个非常关键的误解，也是导致你平衡车无法稳定（摆动、15秒倒下）的核心原因。
1. 纠正转速理解错误
你对参数表的理解存在**“二次减速”**的错误：
图片中的“空载转速 350±20rpm”： 指的是减速箱输出轴（就是那个白色塑料轴）的最终转速。厂家已经把减速比计算在内了。
你的计算逻辑： 你可能认为电机转速是 350，然后还要除以 48（减速比），得到 350÷48≈7.3事实： 内部原始马达的转速其实高达 350×48≈16800 rpm
结论： 你的代码中 #define RPM_MAX 7.3f 是错误的，应该改为 350（或者保守点写 300）。
