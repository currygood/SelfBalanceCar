## 平衡车调试笔记：机械中值与溜车补偿逻辑

### 1. 核心定义（前提条件）

在讨论逻辑前，必须明确你的系统定义：

**坐标系**:
- Y 轴正方向 = 小车前倾（角度 Actual 变大）

**电机极性**:
- PWM 为正数 = 小车向前跑

**控制公式**:
```
PWM = Kp × (Target - Actual)
```

**比例系数**:
- Kp 为负数（例如 -90）

### 2. 现象推导：为什么会"往前溜"？

**现象描述**: 小车站在地上，并没有倒下，但一直在缓慢地向前移动

**逻辑链条**:
```
产生位移：小车向前移动 → 说明电机 PWM 输出为正数
公式推导：根据 PWM = (-90) × (Target - Actual)，要让结果为正数，
         括号里的值必须为负数
结论：(Target - Actual) < 0 ⟹ Target < Actual
```

**本质原因**:
小车当前的物理角度（Actual）比你设定的中值目标（Target）要大。
平衡车控制器认为："我现在处于前倾状态，我快要倒了！我必须赶紧往前跑，去接住我的重心！"

### 3. 深度误解：为什么不能减小 Target？

**直觉误区**: 新手常认为"既然车往前跑，说明前倾了，那我就把目标值改小一点，让它往后仰"

**逻辑纠正**: 如果你把 Target 调小（比如从 0.75 改到 0.5），那么 (Target - Actual) 的值会变得更小（更负）

**结果**:
```
PWM = (-90) × (更小的负数) = 更大的正数
```

**后果**: 小车会往前溜得更快！

### 4. 正确做法：去"匹配"实际角度

要让车停下，目标是让 PWM = 0，即让 Target - Actual = 0

既然现在 Actual（实际角度）偏大，导致了电机前转，你必须主动调大 Target（机械中值），去"追随"并"匹配"那个真实的物理平衡位置

**操作准则**（当 Y+ 为前，且 Kp 为负时）:
- 向前溜车 ⟹ 说明目标值设小了 ⟹ 调大 MECHANICAL_MEDIAN
- 向后溜车 ⟹ 说明目标值设大了 ⟹ 调小 MECHANICAL_MEDIAN

### 5. 形象类比：指尖立棍

想象你在手指上立一把扫帚：
- 扫帚向前方倾斜了，你的手必须向前方移动去接它
- 如果你发现你的手一直带着扫帚向前方走，停不下来
- 这说明你的手一直落后于扫帚的重心
- 为了停下来，你的手必须再往前移动一点，直到彻底移动到重心的正下方

在程序里，"手往前移一点"对应的就是增大那个前倾的目标角度值

### 6. 调试口诀

```
坐标正向是前倾，负号比例记在心。
车往哪溜往哪加，目标匹配才平衡。
```

**注**: 往哪溜往哪加，指的是往坐标系正方向溜，就加目标值；往负方向溜，就减目标值
